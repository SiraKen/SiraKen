GitHub OrganizationでVerifyする

こんにちは。Novalumoでは主にGitHubを使ったバージョン管理をしています。その中でGitHubのOrganizationアカウントを使っているのですが、Organizationといえばこんなマークを見かける人も多いでしょう。

緑色でマークが付いています。これが付いているだけで公式感があって良いですよね。この“Verified”を付与してもらうには「ドメインの検証」を行う必要があります。詳しくは以下のページをご覧ください。

作業を大雑把に説明すると、ドメインの所有を確認するためにDNS設定のTXTレコードに指定された値をセットするというもの。 一度ドメインの検証を行ったのですが、なぜかVerifiedが付きませんでした。どうにか解決すべく色々な部分をチェックすると気になる点が。

そう、Google公式の場合、opensource.googleとgoogle.comのドメイン2つが検証済みになっていました。

実はこれに気づくまではwww.novalumo.llcのみ検証済みになっていましたが、wwwのサブドメイン無しも認証すると…

無事Verifiedになりました！どうやら設定されているWebサイトのURL、メールアドレスのドメインを確認する必要があるみたいです。両方のドメインを登録するとGitHubから確認のメールも届きます。これで完了です。

ドメインが検証済みなのにVerifiedにならない！という方は是非試してみてくださいね！
